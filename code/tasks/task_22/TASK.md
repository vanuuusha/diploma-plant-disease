# Task 22: Практическая реализация — глава 5 диплома

## Статус
in_progress

## Цель

Подготовить материал для главы 5 дипломной работы «Программная реализация информационной системы» — спроектировать и реализовать веб-приложение для диагностики заболеваний пшеницы, использующее лучшие наработки глав 3 и 4 (детектор YOLOv12 + CGFM P5-only из task_16).

## Архитектура

Клиент-серверное веб-приложение:
- **Frontend**: Next.js + Ant Design (русская локаль, зелёная темизация)
- **Backend**: FastAPI + SQLAlchemy + Ultralytics
- **Database**: PostgreSQL
- **LLM**: Gemma 3-27B IT через OpenRouter (рассуждения + комментарии по регионам и снимку)
- **Deployment**: Docker Compose

## Pipeline

1. **Бинарный фильтр** (по документации — ConvNeXt Tiny из НИР1, точность 89%): пшеница? здорово? Останавливает пайплайн, если не пшеница или здорово.
2. **Детектор YOLOv12 + CGFM** (task_16 P5-only, mAP@50=0.377): регионы поражения + вероятности по 9 классам.
3. **Ревью регионов** (Gemma, параллельно через `asyncio.gather`): для каждого bbox — crop + контекст + top-k вероятностей → approved list + комментарий.
4. **Boost & normalize**: вероятности approved-классов × 1.3, нормализация.
5. **Общий комментарий** (Gemma): резюме по всему снимку.
6. **Рекомендации**: хардкод справочник по выбранному диагнозу.
