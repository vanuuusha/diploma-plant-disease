# Task 23: Количественное сравнение диффузии и NST — FID и KID

## Зачем это нужно (мотивация для диплома)

Во 2-й главе диплома рассматриваются два генеративных метода расширения редких классов: Neural Style Transfer (NST) и Stable Diffusion img2img. В тексте уже есть **качественное** сопоставление (визуальные примеры, субъективная оценка) и косвенное downstream-сравнение (прирост mAP@50 на `aug_diffusion` против `aug_oversample` у YOLOv12/RT-DETR в главе 3).

Научный руководитель попросил добавить во 2-ю главу **количественное** сравнение самих сгенерированных изображений — без обращения к детектору. Цель — статистически обосновать тезис «диффузия семантически ближе к реальному распределению полевых снимков, чем NST», который в текущей версии диплома держится только на визуальных примерах.

Для этого нужны две классические метрики качества генеративных моделей:
1. **FID (Fréchet Inception Distance)** — расстояние между реальным и синтетическим распределениями в пространстве признаков InceptionV3.
2. **KID (Kernel Inception Distance)** — несмещённая на малых выборках версия той же идеи через MMD с полиномиальным ядром.

Если гипотеза верна, у диффузии обе метрики будут заметно ниже, чем у NST. Числа уйдут в короткую таблицу на 3 строки в разделе 2.X диплома «Количественное сравнение диффузионной и стилизационной аугментации».

## Цель

Посчитать FID и KID между каждым из синтетических наборов (диффузионный, NST v1, NST v2) и реальным train-сплитом полевых снимков пшеницы. Собрать компактную сводную таблицу и одну поясняющую визуализацию.

## Входные данные

- **Real**: `code/data/dataset_final/train/images/` — реальные полевые снимки пшеницы (train-сплит). Использовать все доступные изображения (N ≈ 3–4 тыс.).
- **Diffusion**: `code/results/task_06/diffusion_v2/` — 450 изображений Stable Diffusion 1.5 img2img (strength=0.40, финальный прогон).
- **NST v1**: `code/results/task_06/nst/` — 60 изображений Neural Style Transfer, Adam-оптимизация, 384 px (Gatys формулировка).
- **NST v2**: `code/results/task_06/nst_v2/` — 32 изображения, LBFGS + YUV-вариант (лучшая NST-версия по внутреннему тесту task_06).

Если каких-то директорий нет — использовать то, что есть, и отметить в RESULT.md.

## Шаги

1. **Установить зависимости** (в существующее venv):
   ```bash
   pip install clean-fid
   ```
   Использовать именно `clean-fid` (не `pytorch-fid`) — он фиксирует нюансы ресайза, и значения получатся сопоставимыми с литературой.

2. **Подготовить изображения**:
   - Real и synthetic должны подаваться как директории с PNG/JPG (без подпапок — перенести/разложить если нужно).
   - Рекомендуемое разрешение для FID — 299×299, но clean-fid сам ресайзит, вмешиваться не надо.
   - Если real-снимков значительно больше synthetic — **не** делать subsampling вручную; FID корректно работает при несбалансированных выборках, достаточно 50+ на меньшей стороне.

3. **Посчитать FID** для каждой пары «real ↔ synthetic»:
   ```python
   from cleanfid import fid
   score_diff = fid.compute_fid(real_dir, diffusion_dir, mode="clean", num_workers=4)
   score_nst_v1 = fid.compute_fid(real_dir, nst_v1_dir, mode="clean", num_workers=4)
   score_nst_v2 = fid.compute_fid(real_dir, nst_v2_dir, mode="clean", num_workers=4)
   ```

4. **Посчитать KID** для тех же пар:
   ```python
   kid_diff = fid.compute_kid(real_dir, diffusion_dir)
   kid_nst_v1 = fid.compute_kid(real_dir, nst_v1_dir)
   kid_nst_v2 = fid.compute_kid(real_dir, nst_v2_dir)
   ```
   KID в clean-fid возвращается как среднее ± std по подвыборкам.

5. **(Опционально, если времени много)** Посчитать FID **per-class** для 4 редких классов (fusarium, root_rot, frost_damage, nitrogen_deficiency), где synthetic был реально сгенерирован:
   - Разбить real-директорию по классам (использовать разметку из dataset_final).
   - Разбить synthetic по классам (по имени файла — `{class_name}_diffusion_{i}.png` и т. п.).
   - Посчитать FID отдельно по каждому классу.
   Если выборки внутри класса слишком малы (< 50 изображений) — KID внутри класса будет надёжнее, FID оставить только для всего набора.

6. **Сохранить визуализацию** — один график `fid_kid_comparison.png`:
   - Группированная столбчатая диаграмма: три метода (Diffusion / NST v1 / NST v2) × две метрики (FID, KID × 1000 для масштаба).
   - Подпись осей, легенда, значения на столбцах.

7. **Записать RESULT.md** со сводной таблицей:
   | Метод | N | FID ↓ | KID × 10³ ↓ |

   | Stable Diffusion img2img | ... | ... | ... |

## Что точно НЕ нужно

- Не считать FID/KID между собственными парами (diffusion ↔ NST) — нам важно расстояние до реального распределения, а не между методами.
- Не интерпретировать абсолютное значение FID через пороги «<10 идеально» — пороги в литературе заданы для ImageNet-generation, а не для сельскохозяйственных снимков. Важна **относительная** разница между методами.
- Не использовать PIL/Inception вручную — clean-fid покрывает все нюансы.
