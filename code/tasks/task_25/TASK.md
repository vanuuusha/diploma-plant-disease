# Task 25: Метрики дисбаланса на уровне изображений (в дополнение к bbox-уровню)

## Статус
pending

## Зачем это нужно (мотивация для диплома)

В главе 2 диплома уже приведены три метрики дисбаланса классов — **IR**, **CV**, **H_norm** — но посчитаны они на уровне **аннотаций (bbox)**. Итоговое значение $IR = 6.18\times$, $CV \approx 0.627$, $H_{norm} \approx 0.93$.

Одновременно в тексте подчёркнут важный нюанс: на уровне **изображений** распределение классов существенно ровнее, чем на уровне bbox. Это объясняется характером проявлений — физиологические патологии (например, недостаток фосфора) дают 10–20 bbox на кадр, а локализованные грибные инфекции — 1–3 очага. В текущей редакции эта разница сформулирована словесно («при соизмеримом числе снимков 440–529 число bbox различается в разы»), но численно она не подкреплена.

Задача — посчитать те же три метрики (IR, CV, H_norm) на **уровне изображений** (сколько кадров приходится на каждый класс, где за класс изображения берётся преобладающий класс bbox-ов на кадре) и вывести рядом с уже имеющимися bbox-метриками. Это даёт количественное подтверждение тезиса про расхождение двух уровней дисбаланса и позволяет осмысленно обосновать выбор стратегии балансировки (oversampling на уровне изображений, аугментация редких классов и т. д.).

## Цель

Для итогового датасета (4443 изображения, 15 867 аннотаций, 9 классов, после фильтрации) посчитать:

1. Таблицу «Класс → число изображений, где класс является преобладающим» (как дополнение к уже существующей таблице «Класс → число аннотаций»).
2. Метрики $IR$, $CV$, $H_{norm}$ на уровне изображений.
3. Сравнительную таблицу bbox-level vs image-level значений всех трёх метрик.

## Входные данные

- Разметка: `code/data/dataset_final/*/labels/*.txt` — YOLO-формат, по одной строке на bbox, первое число — id класса.
- Меппинг id → имя класса: `code/data/dataset_final/data.yaml` (секция `names`).
- Проверить и tran, и test-сплиты (либо объединить их для целей главы 2 — там метрики считаются по всему итоговому датасету целиком).

## Шаги

1. **Собрать статистику по изображениям**. Для каждого `.txt`-файла разметки:
   - прочитать все bbox;
   - определить **преобладающий класс** на кадре (по количеству bbox; при равенстве — по суммарной площади bbox);
   - присвоить изображение этому преобладающему классу.

2. **Построить таблицу N_images_per_class**. Вывести также минимум и максимум — ожидаемо числа в диапазоне 400–530 (для проверки сходимости с уже приведённой в дипломе оценкой).

3. **Посчитать три метрики на уровне изображений**:

   $$IR_{img} = \frac{\max_c N^{img}_c}{\min_c N^{img}_c}$$

   $$CV_{img} = \frac{\sigma_{N^{img}}}{\mu_{N^{img}}}$$

   $$H^{img}_{norm} = \frac{-\sum_c p^{img}_c \log p^{img}_c}{\log K}$$

   где $N^{img}_c$ — число изображений с преобладающим классом $c$, $p^{img}_c = N^{img}_c / \sum_c N^{img}_c$, $K = 9$.

4. **Перепроверить bbox-level числа** (должны совпасть с уже имеющимися): $IR_{bbox} = 6.18$, $CV_{bbox} \approx 0.627$, $H^{bbox}_{norm} \approx 0.93$. Если не совпадут — разобраться в причине (возможно, изменился состав датасета или разметка).

5. **Сформировать сводную таблицу**:

   | Метрика | На уровне bbox | На уровне изображений |
   |---------|---------------:|----------------------:|
   | IR | 6.18× | ... |
   | CV | 0.627 | ... |
   | $H_{norm}$ | 0.93 | ... |

6. **Опционально** — построить диаграмму, где слева столбики «число аннотаций на класс», справа «число изображений на класс». Это наглядно покажет расхождение двух уровней и пригодится как дополнительный рисунок в главе 2.
