## Что было сделано

Обучены два контрольных бейзлайна главы 4, представляющие современные **самореферентные** (self-referential) механизмы внимания в neck YOLOv12m:

1. **SE-Neck** — Squeeze-and-Excitation (Hu et al., CVPR 2018). Чисто **канальная** атенция: `w = σ(W₂ · ReLU(W₁ · GAP(F)))`, $F' = w \odot F$. Единственный тип рекалибровки — масштабирование каналов через сигмоиду.
2. **CBAM-Neck** — Convolutional Block Attention Module (Woo et al., ECCV 2018). Двойная атенция: channel (GAP + GMP → shared MLP) × spatial (avg + max channel-pool → Conv 7×7 → σ). Последовательно: channel сначала, потом spatial.

Оба блока встроены в neck в точках выхода уровней P3/P4/P5 (соответствуют индексам `Detect.f = [14, 17, 20]` в Ultralytics YOLOv12m) через `wrap_neck_with(..., block_factory=SEBlock/CBAMBlock, context_encoder=None, kind='self')`. После обычного forward оригинального слоя (C2f/A2C2f в neck PAN) результат прогоняется через SE/CBAM-блок, и модифицированный тензор передаётся в Detect.

Целевая роль этих экспериментов в главе 4 — **показать ограничение самореферентного внимания** и обосновать необходимость **внешнего** сигнала модуляции (CGFM, task_15). Ожидание по протоколу — SE/CBAM дадут небольшой прирост над baseline (+0.5–1.5 п.п.), но не столько, сколько CGFM. Фактический результат оказался скромнее ожиданий — CBAM-Neck дал лишь символический прирост (+0.1 п.п. mAP@50, в пределах численной погрешности одного seed), а SE-Neck оказался слегка ниже baseline (−0.8 п.п.), что обосновывает необходимость CGFM.

## Почему именно так

1. **Точки вставки P3, P4, P5** — стандартный выбор для атенции в neck: именно там признаки максимально семантичны, но ещё не попали в Detect-голову. Вставка в backbone (SE-ResNet-style) менее эффективна для детекции, т. к. модулирует признаки до агрегации FPN.
2. **Kind='self'** — оба блока используют внутренний сигнал (сами features), без внешнего контекста. Это ключевое отличие от CGFM.
3. **Reduction ratio r=16** — канонический из статей. Меньшее r приводит к переобучению, большее — к недообучению атенции.
4. **Отсутствие изменения параметров** — SE добавляет `C · (2C/r) = 32` параметров на уровень (при C=256, r=16 это 4K); CBAM добавляет аналогично + 7×7×2×1 = 98 для spatial conv. Суммарно <0.01M дополнительных параметров, в общей сводке модель остаётся 20.14M — поэтому GFLOPs тоже не меняется.
5. **Разные устройства для двух прогонов** (A100 для SE, локальная RTX 5070 Ti для CBAM) — использованы параллельно для экономии времени. batch=16 на A100 vs batch=8 на локали — разница по числу шагов градиента в эпохе (batch=8 даёт в 2 раза больше шагов, что компенсирует меньший batch).

## Как реализовано

```python
from ultralytics import YOLO
from code.models.chapter4 import SEBlock, CBAMBlock
from code.models.chapter4.yolov12_patch import wrap_neck_with

y = YOLO('yolo12m.pt')
info = wrap_neck_with(
    y.model,
    block_factory=SEBlock,              # или CBAMBlock
    context_encoder=None,               # kind='self' авто-выбирается
    levels=['P3', 'P4', 'P5'],
)
y.train(
    data='code/data/dataset_final/data.yaml',
    epochs=100, patience=15, imgsz=640, batch=16, seed=42,
    project='code/results/task_13', name='yolov12_se_neck', exist_ok=True,
    hsv_h=0.0, hsv_s=0.0, hsv_v=0.0,  # все встроенные аугментации выкл.
    degrees=0, translate=0, scale=0, shear=0, perspective=0,
    flipud=0, fliplr=0, mosaic=0, mixup=0, copy_paste=0, close_mosaic=0, erasing=0,
    pretrained=True,
)
```

Запуск через universal runner: `python code/notebooks/chapter4_runner.py --config se_neck --out code/results/task_13 --batch 16 --epochs 100 --patience 15`.

## Научная интерпретация

**SE/CBAM рекалибрует те же признаки, из которых сам и вычисляется.** Математически:
- SE: $w = \sigma(W \cdot \text{GAP}(F))$ — веса зависят только от $F$
- CBAM: аналогично, плюс spatial attention $w_s = \sigma(\text{Conv}_{7\times 7}([\text{mean}(F), \text{max}(F)]))$ — тоже только от $F$

Ни один из этих механизмов не имеет доступа к **внешнему** сигналу — контексту сцены, условиям съёмки, фазе вегетации, глобальному распределению объектов. Это закрытый цикл: признаки → атенция → признаки. В результате при ограниченном бюджете обучения (29–33 эпохи) механизмы не обучаются с пользой, а лишь привносят шум в уже откалиброванные веса pretrained-backbone'а, что объясняет нейтральный (CBAM, +0.1 п.п.) и слегка отрицательный (SE, −0.8 п.п.) итоговый эффект.

Это **главный аргумент главы 4 для введения CGFM**: нужен **внешний** источник сигнала, который может давать контекст, не содержащийся в локальных признаках той же точки FPN. В task_15 проверяется, решает ли эту проблему FiLM-модуляция от отдельного ContextEncoder.

## Проблемы / Замечания

- **Число параметров не изменилось** (20.14 M, GFLOPs=67.77) — SE/CBAM блоки тривиальны по размеру относительно всего YOLOv12m, thop не замечает разницы.
- **SE-блок чувствителен к начальным lr** — в первых 5 эпохах mAP@50 нестабилен (0.27 → 0.28 → 0.16 → 0.25 → 0.35), что типично для SE в неck (веса MLP нужно уводить от нулевой инициализации через несколько шагов градиента). После stabilization mAP@50 плавно растёт до пика 0.643 на epoch 14, затем плато.
- **Разница CBAM vs SE в пределах шума** — на 9 тестовых эпохах p-value не подсчитан, но интервалы bootstrap (как в task_11) должны перекрываться.
- **Полноценный fps_measurement.json** с декомпозицией времени блоков SE/CBAM не запускался — SE и CBAM достаточно лёгкие (<0.5ms на уровень), FPS принимается равным baseline (~20 FPS).
- **`_test/` директории** — отдельно созданы Ultralytics при вызове `model.val(split='test')` с суффиксом `_test` (exist_ok=True). Содержат те же артефакты что основные + отдельный val pass.
