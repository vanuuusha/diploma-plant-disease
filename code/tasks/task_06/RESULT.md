После task_04 (классическая аугментация) и task_05 (агрессивный oversampling, закрывший 70% дефицита) у 4 редких классов остаётся недобор ~800 bbox до целевого уровня 80% медианы. ## 2. Два сравниваемых подхода

### 2.1 Stable Diffusion img2img
**Модель**: `stable-diffusion-v1-5/stable-diffusion-v1-5` (community-mirror оригинального Runway SD-1.5, удалённого с HuggingFace). Архитектура — **latent diffusion** (Rombach et al. 2022, *High-Resolution Image Synthesis with Latent Diffusion Models*).

**Принцип работы img2img**:
1. VAE-энкодер сжимает реальное изображение (seed) в латент 64×64×4.
2. К латенту добавляется гауссов шум, пропорциональный параметру `strength` (0 — без шума, 1 — чистый шум).
3. U-Net итеративно денойзит латент обратно, условно обусловленно текстовым промптом через CLIP-энкодер.
4. VAE-декодер превращает латент в RGB.

Математика: прямой процесс добавления шума — цепь Маркова
$$q(x_t | x_{t-1}) = \mathcal{N}(x_t; \sqrt{1 - \beta_t} \, x_{t-1}, \beta_t \mathbf{I})$$
Обратный процесс (обучаемый) — предсказание шума U-Net'ом и денойзинг.

**Ключевые параметры нашего прогона**:
- `strength = 0.40` (фиксировано) — сохраняет ~60% латента, композиция остаётся. Это критично для наследования меток от seed.
- `num_inference_steps = 30` — стандартный баланс качество/скорость.
- `guidance_scale = 7.5` — стандартный CFG.
- `torch_dtype = fp16` — помещается в 5 ГБ VRAM.

### 2.2 Neural Style Transfer (Gatys et al., 2016)
**Принцип**: передать текстурные характеристики одного изображения (style — больной лист с симптомами) на композицию другого (content — здоровый/другой лист), через оптимизацию пикселей целевого изображения.

**Математика**:
- Через замороженную VGG19 пропускаем content, style и target.
- **Content loss** на слое conv4_2:
  $$L_{content} = \sum_l \| F^{target}_l - F^{content}_l \|^2$$
- **Style loss** — расхождение матриц Грама на 5 слоях {conv1_1, conv2_1, conv3_1, conv4_1, conv5_1}:
  $$G^l_{ij} = \sum_k F^l_{ik} F^l_{jk}$$
  $$L_{style} = \sum_l \| G^{target}_l - G^{style}_l \|^2$$
- Итоговая функция: $L = \alpha L_{content} + \beta L_{style}$ при $\beta/\alpha = 10^6$.
- Оптимизация **самого изображения target** (его пикселей), веса VGG заморожены.

**Мы реализовали две версии NST**, чтобы быть честными в сравнении:
- **v1** (наивная): Adam, lr=0.02, 250 шагов, 384×384.
- **v2** (классическая): **L-BFGS** (как в оригинальной статье Gatys), 512×512, 300 шагов, + **color preservation** через YUV (сохраняем хроматику content, меняем только яркостную компоненту), + total-variation regularization для гладкости.

## 3. Визуальное сравнение

**Основной грид**: `code/results/task_06/MASTER_COMPARISON.png` — 4 строки (по редкому классу) × 4 колонки (Seed / Diffusion / NST v1 / NST v2).

### Наблюдения по классам

**Корневая гниль**:
- Diffusion: сохранена композиция стеблей пшеницы, добавлены правдоподобные бурые пятна. Выглядит как реальное фото из того же ряда съёмки.
- NST v1: вертикальные оранжево-бурые «мазки», структура листа не читается.
- NST v2: серебристо-зелёные щетинистые текстуры, симптомов корневой гнили не видно.

**Септориоз**:
- Diffusion: чёткий бурый очаг с сетчатой некротической структурой — **правильный** симптом.
- NST (обе версии): вертикальные штрихи по всему листу; симптом как локализованная болезнь не воспроизведён.

**Недостаток N**:
- Diffusion: умеренное пожелтение, реалистично.
- NST v2: **сине-бирюзовый лист** с оранжевыми точками. Цвет катастрофически неверный (color preservation не справился с доминантой style), симптом — ржавчина, а не дефицит азота.

**Заморозки**:
- Diffusion: зелёное поле пшеницы (модель недоинтерпретировала абстрактное понятие «frost damage»).
- NST v2: в правом нижнем углу **иероглифы** из водяного знака style-изображения. Критический артефакт — NST переносит всё, что есть в style, включая watermark.

### Диагноз: почему NST принципиально проигрывает

| Критерий | Diffusion img2img | NST (v1/v2) |
|----------|-------------------|-------------|
| Реализм | Фотореалистичный (априор LAION-5B) | Художественный фильтр |
| Локализация симптомов | Может разместить пятно | Размазывает стиль равномерно |
| Семантическое понимание | Текст → что генерировать | Только статистика Грама, без смысла |
| Цветовая точность | Сохраняет натуральную палитру | Утечка цвета из style |
| Устойчивость к артефактам style | Не переносит watermark/подписи | Переносит всё |
| Валидность bbox | При strength≤0.45 композиция цела | Позиционно валидна, но симптомы не в тех местах |
| Скорость | ~5 с/image (fp16) | 10–30 с/image (LBFGS 300 шагов) |

**Корневая причина**: матрицы Грама описывают **статистику текстуры**, лишённую пространственной информации. Реальные симптомы болезней **локальны и структурированы** — пятно на листе имеет чёткие границы, контекст (окружающая здоровая ткань), часто диагностическую сетку. NST не может воспроизвести ничего из этого. Stable Diffusion через CLIP-conditioning имеет доступ и к семантике (слово «septoria» активирует известные модели симптома), и к пространственной информации (через U-Net attention).

### Почему даже NST v2 (улучшенный) не спас ситуацию
Мы сделали всё, что рекомендуется в литературе:
- **L-BFGS** вместо Adam — это оптимизатор из оригинальной статьи Gatys, сходится быстрее и точнее.
- **Color preservation через YUV** — Yuan et al. 2017 показали, что это помогает сохранять «тон» content.
- **Higher resolution** (512 vs 384) — больше деталей.
- **Total variation regularization** — гладкие результаты, меньше визуального шума.

Это улучшило качество по сравнению с v1 (меньше мазков, лучше сохранён контент), но **не решило фундаментальную проблему** отсутствия семантики. Диагностическое правило: если метод не знает *что именно* он рисует, он не может сделать это правильно.

Дальнейшие улучшения NST, которые МОГЛИ бы помочь (не реализовывали):
- **Masked Style Transfer** — применять NST только внутри bbox симптома.
- **Semantic NST** (Luan et al., 2017) — с картами сегментации, стиль переносится только между соответствующими регионами.
- **Neural Neighbor Style Transfer** (2022) — использует сопоставление патчей по смыслу.

Все они концептуально ближе к генеративным моделям, чем к оригинальному NST. В нашем пайплайне они не нужны, так как Diffusion уже решает задачу.

## 4. Расчёт количества синтетических изображений

**Не генерируем фиксированное число** — количество рассчитывается на класс от его **дефицита**:

```
gap[c]          = 0.8 × median − count_after_aug[c]     # задумка task_05+task_06
diff_need[c]    = 0.3 × gap[c]                           # доля, закрываемая Diffusion
num_images[c]   = ceil(diff_need[c] / avg_bbox_per_img[c])
```

`avg_bbox_per_img[c]` = (всего bbox класса в исходном датасете) / (число изображений, где класс присутствует).

Результат расчёта:

| Класс | count_aug | final target | gap | 30% gap (bbox) | avg_bbox/img | **Сгенерировано** |
|-------|----------:|-------------:|----:|---------------:|-------------:|------------------:|
| Корневая гниль | 2089 | 2556 | 467 | 140 | 1.93 | **73** |
| Септориоз | 2073 | 2556 | 483 | 145 | 2.03 | **72** |
| Недостаток N | 1802 | 2556 | 754 | 226 | 1.73 | **131** |
| Повреждение заморозками | 1568 | 2556 | 988 | 296 | 1.70 | **174** |
| **Итого** | | | | **807** | — | **450** |

**Интерпретация**: Заморозки получают больше всего диффузии (174 image), потому что у них самый большой дефицит. Корневая гниль и Септориоз — меньше (по ~72), потому что они ближе к цели.

## 5. Seed selection — почему из оригинальных, не из аугментированных

В v2-скрипте фильтр `if "_aug" in stem or "_bal" in stem: continue`. Почему:

- Цепочка «оригинал → augment → diffusion» ввела бы двойную регуляризацию: аугментация исказила бы исходник, диффузия — ещё раз. Результат был бы дальше от реальности, чем нужно.
- Оригинальные снимки — «чистый» источник ракурса и композиции. Диффузия лишь добавляет синтетический слой «симптома», оставляя остальное реальным.

## 6. Почему метки наследуются от seed

При `strength = 0.40`:
- Добавляется относительно малый шум к латенту.
- Денойзинг может изменить текстуры (добавить пятна), но **пространственная композиция** (положение стебля, угол листа, фон) сохраняется с точностью ~60–70%.
- Это означает, что bbox, размеченный на seed-изображении, будет **почти** в том же месте и в synthetic (смещение может быть 10–20 px для разрешения 512×512).

Для детекции при обучении такое приближение валидно: детектор толерантен к небольшим неточностям меток (IoU threshold обычно ≥0.5). Но для **тонкой оценки** требуется ручная переразметка — это отмечено в разделе «Ограничения».

**Альтернативы, от которых отказались**:
- *Наследовать только whole-image bbox* (1 box на весь снимок класса target) — слишком грубая разметка, теряется информация о локализации симптома.
- *Использовать inpainting* (генерация только внутри bbox-маски seed) — корректнее, но требует маски; при polygon→bbox конвертации мы эти маски уже потеряли. Для будущей работы — return to segmentation masks.

## 8. Ограничения и известные проблемы

1. **Метки синтетики унаследованы** — приблизительны. Для обучения YOLO это нормально (IoU threshold толерантен), но для метрики на «чистых» синтетических данных — смещение.

2. **SD-1.5 не обучена на аграрных снимках**. Промпты на специализированные термины («septoria leaf spot») интерпретируются через общее «представление о болезнях растений» из LAION. Дальнейший путь — **LoRA** (Low-Rank Adaptation) на 100–500 реальных снимках каждого класса. LoRA добавляет ~40 МБ параметров, обучается 30–60 минут на нашей GPU, и существенно улучшает «доменное соответствие».

3. **Заморозки плохо воспроизводятся**. «Frost damage» на пшенице — неочевидный визуальный признак (белёсые края, некротические пятна), SD-1.5 рисует просто зелёную пшеницу. Здесь промпт-инженирование имеет потолок, нужен fine-tune.

4. **450 изображений = +0.8% аннотаций к train**. Масштаб **демонстрационный**. Для полноценного влияния на метрики обучения нужно порядка 2000–5000 синтетических снимков. Это доступно (просто время вычислений), но выходит за рамки task_06 — оставлено как направление для финального эксперимента главы 4.

`synth_manifest.json` — промежуточный формат: лист записей `{cls_id, class_name, seed_image_path, seed_label_path, out_image_path, strength, prompt}`. Этот формат позволяет:
- фильтровать генерации по критериям (напр., `strength ≤ 0.55`),
- переиспользовать одни и те же изображения в разных стратегиях сборки,
- вести аудит происхождения каждого синтетического файла.
