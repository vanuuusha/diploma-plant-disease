# Task 06: Генеративные аугментации — Diffusion + Neural Style Transfer для редких классов

## Статус
pending

## Цель
Сгенерировать синтетические изображения для редких классов двумя методами:
1. **Diffusion-аугментация** — генерация изображений больных листьев через Stable Diffusion (img2img / ControlNet / inpainting)
2. **Neural Style Transfer (NST)** — перенос стиля (симптомов болезни) с больных листьев на здоровые

Это экспериментальные методы для главы 2 диплома. Нужны как сами изображения, так и визуальные примеры для текста.

## Зависимости
Выполнить после task_03 (EDA — чтобы знать какие классы редкие). Не зависит от task_04/05 — это параллельный эксперимент.

## Входные данные
- `code/data/dataset/train/` — оригинальный train сплит
- Результаты task_03 — список редких классов и их число аннотаций

## Часть 1: Diffusion-аугментация

### 1.1 Подход
Использовать **Stable Diffusion** (через `diffusers` библиотеку) в режиме **img2img**:
- Взять реальное изображение больного листа как входное
- Промпт описывает нужный класс болезни: `"wheat leaf with {disease_name} symptoms, close up, agricultural field, natural lighting"`
- `strength` = 0.4–0.6 (сохраняем структуру исходного изображения)
- Генерировать по 5–10 вариантов для каждого редкого класса

Альтернативный подход (если img2img даёт плохое качество):
- Text-to-image с промптом, описывающим болезнь
- Промпты подготовить заранее для каждого класса

### 1.2 Модель
- `stabilityai/stable-diffusion-2-1` или `runwayml/stable-diffusion-v1-5`
- Если GPU-память < 8GB, использовать `float16` + `enable_model_cpu_offload()`

### 1.3 Для каждого редкого класса
- Выбрать 3–5 реальных изображений как seed
- Сгенерировать 5 вариантов от каждого seed
- Итого: 15–25 синтетических изображений на редкий класс

### 1.4 Ручная фильтрация (автоматическая)
- Вычислить CLIP-score между промптом и результатом (если доступен)
- Или просто сохранить ВСЕ, а фильтрацию оставить для ручного просмотра
- Пометить сгенерированные файлы: `{class_name}_diffusion_{i}.png`

## Часть 2: Neural Style Transfer

### 2.1 Подход
Перенос стиля симптомов болезни с больного листа на здоровый:
- **Content image** — здоровый лист пшеницы (из датасета или отдельных фото)
- **Style image** — лист с ярко выраженными симптомами нужного класса
- Результат: изображение, сохраняющее структуру здорового листа, но с «наложенными» симптомами

### 2.2 Реализация
Использовать **PyTorch NST** (Gatys et al., 2016) или быстрый вариант через `torchvision.models.vgg19`:
- VGG19 для извлечения признаков
- Content loss на conv4_2
- Style loss на conv1_1, conv2_1, conv3_1, conv4_1, conv5_1
- Оптимизация 300 шагов, Adam, lr=0.01
- content_weight=1, style_weight=1e6

Альтернатива: если есть `fast-neural-style` (Johnson et al.) — использовать его для скорости.

### 2.3 Для каждого редкого класса
- Выбрать 3 стилевых изображения (яркие симптомы)
- Выбрать 5 контентных изображений (здоровые листья)
- Сгенерировать 15 вариантов (3 стиля × 5 контентов)
- Пометить: `{class_name}_nst_{i}.png`
