# Скилл: Сохранение артефакта решения

Когда пользователь вызывает `/save_art`, ты должен:

## 1. Определить следующий номер артефакта

Прочитай содержимое папки `диплом/artifacts/` и найди максимальный номер существующего артефакта (файлы имеют формат `ART_NNN_НАЗВАНИЕ.md`). Следующий артефакт получает номер `NNN + 1`.

## 2. Собрать выжимку из текущего обсуждения

Перечитай текущий диалог и сформируй **выжимку** ключевого решения/договорённости. Выжимка должна содержать:

- **Суть в одном абзаце** — что решено
- **Проблема** — какую проблему решает
- **Решение** — конкретное описание (формулы, схемы, конфигурации — если есть)
- **Аргументация** — почему именно так, с чем сравнивали, что отвергли
- **Связи** — ссылки на другие артефакты, если есть зависимости

## 3. Спросить у пользователя

Перед сохранением покажи пользователю:
- Предлагаемое название файла (например `ART_002_SEVERITY_LOSS.md`)
- Краткое содержание (3-4 строки)
- Спроси подтверждение

## 4. Сохранить файл

Записать файл в `диплом/artifacts/ART_NNN_НАЗВАНИЕ.md` в формате:

```markdown
# ART-NNN: Краткое название

> **Тип:** [Решение / Концепция / Архитектура / Эксперимент / ...]
> **Дата:** YYYY-MM-DD
> **Статус:** Принято | Обсуждается | Отвергнуто
> **Контекст:** Краткое описание контекста обсуждения

---

[Содержание выжимки]
```

## 5. Подтвердить сохранение

Показать пользователю ссылку на сохранённый файл.

## Важно

- Артефакт — это **фиксация решения**, а не черновик. Он должен быть самодостаточным: человек, читающий его в новом чате, должен полностью понять что решено и почему.
- Не дублируй артефакты — если решение уже зафиксировано, обнови существующий артефакт.
- Пиши по-русски, техтермины — двуязычно (русский + английский в скобках).
