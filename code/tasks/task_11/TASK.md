# Task 11: Сводный анализ главы 3 — кросс-сравнение 4 детекторов × 4 датасетов

## Статус
pending (стартует после завершения task_07, task_08, task_09, task_10)

## Цель

Собрать результаты всех 16 экспериментов из task_07–task_10 в единую аналитическую картину, обосновать выбор базовой архитектуры для главы 4 (CGFM) и подготовить числовой и визуальный материал для раздела 3.2 диплома.

## Шаги

1. **Создать ноутбук** `code/notebooks/chapter3_summary.ipynb`.
2. **Собрать данные**: прочитать 4 `summary.csv` (по одному из каждой task_07..task_10), объединить в одну длинную таблицу (детектор × вариант × метрики).
3. **Построить сводную матрицу 4 × 4** — mAP@50 (и аналогичные для mAP@50-95, Precision, Recall, FPS). Строки — детекторы, колонки — варианты датасета.
4. **Построить heat-map** прироста от исходного baseline по каждой метрике (Δ mAP@50 относительно baseline для каждого детектора в каждой стадии аугментации).
5. **Построить диаграмму «FPS vs mAP@50-95»** — scatter-plot по всем 16 точкам (4 детектора × 4 варианта), цвет — детектор, форма — вариант. Наглядно показывает трейдофф скорость/точность.
6. **Построить столбцовые диаграммы прироста по классам**: для каждого детектора — 9 классов × 4 варианта датасета (вклад аугментации в редкие классы).
7. **Сравнительная качественная галерея**: собрать из `predictions_examples/` всех 16 запусков изображения на одних и тех же тестовых файлах (из `code/docs/chapter3_qualitative_sample.txt`) в единую сетку (например, 4 строки на изображение = 4 детектора, для варианта aug_diffusion).
8. **Статистический тест различий**: для пары «лучший детектор на aug_diffusion» vs «второй» — оценить, значима ли разница. Варианты: bootstrap доверительного интервала mAP@50 по test (1000 раз с возвратом выбрать подмножество изображений, пересчитать mAP@50, получить распределение). Записать 95 % доверительные интервалы для топ-2 детекторов.
9. **Финальная таблица для главы 3** — топ-1 результат каждого детектора (на aug_diffusion) + FPS + число параметров + размер модели.
10. **Выбор архитектуры для главы 4**: обоснование выбора в свободной форме в `RESULT.md` (3–5 абзацев). Критерии: максимум mAP@50-95 при FPS ≥ 30, простота интеграции CGFM-модуля (YOLOv12 имеет явный neck, куда легко встраивается FiLM).
