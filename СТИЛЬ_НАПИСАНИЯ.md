# Инструкция по стилю написания текста

> Данный файл описывает стиль, тон и приёмы написания текста, выявленные из двух КНИР автора (НИР1 и НИР2). При генерации нового текста для дипломной работы необходимо следовать всем пунктам ниже, чтобы обеспечить стилистическое единообразие с ранее написанными работами.

---

## 1. Общий тон и регистр

- **Академический научно-технический стиль** — формальный, безличный, без разговорных элементов.
- Тон серьёзный, но не перегруженный канцеляризмами. Автор пишет доступно для специалиста в области ИТ, но сохраняет научную строгость.
- **Никогда** не используется первое лицо единственного числа ("я сделал", "я исследовал").
- Используется:
  - Безличная форма: «был разработан», «было проведено», «рассматривается».
  - Пассивный залог: «модель была обучена», «результаты были получены».
  - Третье лицо: «автор провёл» — **не используется**. Вместо этого: «в работе проведено», «в рамках исследования выполнено».
  - Формулировки от лица работы: «настоящая работа посвящена», «в данном разделе приводится».

---

## 2. Структура предложений

### 2.1 Длина и сложность
- Предложения **длинные и многосоставные**, часто 30–60 слов. Это характерная черта стиля — не бояться длинных конструкций.
- Активно используются **причастные и деепричастные обороты** для уплотнения информации в одном предложении.
- Типичная структура: основное утверждение → уточнение в скобках → следствие или вывод.

### 2.2 Примеры характерных конструкций
- «Одной из наиболее существенных и широко признанных проблем в области применения глубокого обучения для диагностики заболеваний растений является дефицит обширных и качественно аннотированных наборов обучающих данных.»
- «Данное направление включает в себя задачу обнаружения объектов (object detection), заключающуюся в идентификации и выделении патологических областей с помощью ограничивающих рамок (bounding boxes), и задачу сегментации изображений (image segmentation).»
- «Ключевыми вызовами на данном этапе являются значительная межклассовая схожесть визуальных симптомов различных заболеваний и высокая внутриклассовая вариативность проявлений одного и того же патологического процесса, обусловленная стадией развития болезни, условиями окружающей среды и генотипом растения.»

### 2.3 Вводные и связующие конструкции
Часто используемые обороты (применять регулярно для связности):
- «В связи с этим...»
- «Вместе с тем...»
- «Наряду с...»
- «В отличие от...»
- «Таким образом...»
- «Следует отметить, что...»
- «Важно подчеркнуть, что...»
- «Однако...»
- «Также необходимо учитывать...»
- «Данный подход позволяет...»
- «С учётом данного контекста...»
- «Эти факторы подчёркивают актуальность...»

## 3. Терминология и лексика

### 3.1 Двуязычная терминология
- При первом упоминании термина: русский вариант + английский в скобках.
  - «обнаружение объектов (object detection)»
  - «ограничивающие рамки (bounding boxes)»
  - «средняя прецизионность (mean Average Precision, mAP)»
- При повторном использовании — только русский ИЛИ только аббревиатура (mAP, IoU).
- Английские названия моделей и фреймворков оставлять как есть: YOLOv10, Faster R-CNN, ResNet, PyTorch, Ultralytics.

### 3.2 Характерная лексика (использовать активно)
- «робастность» (не «устойчивость» в контексте моделей)
- «генерализация» (не «обобщение»)
- «инференс» (не «вывод модели»)
- «пайплайн» (не «конвейер»)
- «датасет» (не «набор данных», хотя оба варианта встречаются)
- «аугментация» (не «расширение данных»)
- «апробация» (не «проверка»)
- «репрезентативность» (не «представительность»)
- «метрики» (не «показатели», хотя оба используются)
- «домен» / «доменный» (в значении «предметная область»)
- «фитопатологический» / «фитосанитарный»
- «каскадный подход»
- «вычислительная эффективность»

### 3.3 Слова-паразиты (НЕ использовать)
- «данный» — используется, но в меру (не в каждом предложении).
- «является» — используется часто, это нормально для научного стиля.
- **НЕ** используются: «на самом деле», «по сути», «грубо говоря», «в целом и общем», «так сказать».

## 4. Структура разделов и абзацев

### 4.1 Типичная структура раздела
Каждый крупный раздел (глава) следует паттерну:
1. **Вводный абзац** — зачем этот раздел нужен, что в нём будет рассмотрено.
2. **Основное содержание** — детальное изложение (подразделы).
3. **Итоговый абзац** — краткое резюме раздела, вывод, связка со следующим разделом.

### 4.2 Типичная структура абзаца
- Абзацы **объёмные**, по 5–10 предложений (иногда больше).
- Структура: тезис → развёрнутое обоснование → примеры/уточнения → мини-вывод.
- Абзацы **не обрываются** — каждый завершается логически.

### 4.3 Переходы между разделами
- Каждый раздел завершается резюмирующим абзацем, который «мостит» к следующему.
- Пример: «Проведённый анализ существующих публичных наборов данных выявил их значительную неоднородность... Сформированный собственный датасет представляет существенный вклад в решение проблемы...»

---

## 5. Работа с перечислениями (списками)

### 5.1 Когда используются списки
- Перечисление задач исследования.
- Перечисление заболеваний, классов, метрик.
- Перечисление преимуществ и недостатков подходов.
- Перечисление этапов (пайплайна, архитектуры).

### 5.2 Формат перечислений
- Элементы списка — **краткие**, по 1–2 предложения.
- Перед списком — вводное предложение с двоеточием.
- Списки часто организованы по паттерну «Описание: ... Преимущества: ... Недостатки: ...»

### 5.3 Паттерн «Преимущества / Недостатки»
Для описания моделей, датасетов, подходов используется устойчивый формат:
```
Название:
Описание: <что это>.
Преимущества: <плюсы>.
Недостатки: <минусы>.
```
Этот паттерн применяется последовательно ко всем сравниваемым объектам.

## 6. Работа с рисунками и таблицами

### 6.1 Ссылки на рисунки
- Всегда есть ссылка в тексте **до** самого рисунка: «Пример показан на рисунке 1», «Графики обучения приведены на рисунке 6».
- Подпись под рисунком: «Рисунок N — Название» (тире, не точка).

### 6.2 Ссылки на таблицы
- Аналогично: «Результаты приведены в таблице 1».

### 6.3 Код
- В НИР2 фрагменты кода вставлены прямо в текст с минимальными комментариями.
- Перед кодом: «Ключевой код для модели:» или «Для обучения используется скрипт:».
- Код — только ключевые строки (10–15 строк), не полный скрипт.

---

## 7. Введение — шаблон

Введение строится по чёткому плану:
1. **Широкий контекст** (1–2 предложения): сельское хозяйство, продовольственная безопасность, современные вызовы.
2. **Проблема** (1–2 предложения): конкретная угроза (заболевания, потери урожая), статистика.
3. **Текущие методы и их ограничения** (1–2 предложения): традиционная диагностика, её недостатки.
4. **Предлагаемое решение** (1–2 предложения): компьютерное зрение, глубокое обучение.
5. **Нерешённые проблемы** (1–2 предложения): дефицит данных, робастность, интерпретируемость.
6. **О чём данная работа** (1–2 предложения): что конкретно исследуется.
7. **Цель работы** (1 предложение, выделено): формулировка цели.
8. **Задачи исследования** (перечисление): 4–5 задач.

---

## 8. Заключение — шаблон

Заключение строится по плану:
1. **Общее резюме** (1–2 предложения): что было сделано в работе.
2. **Подтверждение достижения цели** (1–2 предложения): цель была достигнута, через какие действия.
3. **Детали результатов** (2–3 предложения): конкретные числа (точность, mAP, скорость).
4. **Ограничения и направления развития** (2–3 предложения): что можно улучшить.
5. **Перспективы** (1–2 предложения): куда это ведёт (точное земледелие, мобильные устройства и т.д.).

---

## 9. Описание экспериментов и моделей

### 9.1 Структура описания каждой модели (НИР2)
Для каждой исследуемой модели используется единый шаблон:
1. **Структура модели** — как она устроена (backbone, neck, head).
2. **Преимущества и недостатки** — в контексте задачи.
3. **Применение к задаче** — почему рассматривается, какая гипотеза.
4. **Реализация и код** — ключевые строки кода.
5. **Результаты** — графики обучения, метрики, время инференса.

### 9.2 Подача числовых результатов
- Числа подаются конкретно: «mAP@50=0.78», «210 мс/изображение», «точность 73%».
- Сравнение даётся относительно: «чуть более высокие метрики», «намного опередив».
- После числовых результатов — интерпретация: почему так, что это значит для практики.

## 10. Специфические стилистические особенности

### 10.1 Тире и скобки
- Широко используются тире (длинное «—» и короткое «–»).
- Тире для пояснений: «фитопатологические заболевания – по данным ФАО ООН, они вызывают потери...»
- Скобки для уточнений и английских эквивалентов: «(bounding boxes)», «(например, ResNet, EfficientNet)».

### 10.2 Двоеточие
- Активно используется перед перечислениями и пояснениями.
- «Ключевые ограничения:», «Преимущества:», «Задачи исследования:».

### 10.3 Кавычки
- Для специфических терминов при первом упоминании: «здоровое»/«больное», «объектами».
- Для названий: «МИСИС», «Plantix».
- Используются «ёлочки» — « ».

### 10.4 Аббревиатуры
- При первом употреблении — полная расшифровка + аббревиатура в скобках.
- Далее — только аббревиатура.

## 11. Различия между НИР1 и НИР2

| Параметр | НИР1 | НИР2 |
|----------|-------|-------|
| Характер | Обзорно-аналитическая | Экспериментальная |
| Абзацы | Очень длинные, теоретические | Более компактные, конкретные |
| Фокус | Архитектуры, методы, обзор литературы | Сравнение моделей, метрики, код |
| Списки | Реже, больше текста | Чаще, структурированнее |
| Цифры | Мало конкретных метрик | Много конкретных числовых результатов |
| Код | Отсутствует | Есть фрагменты кода |

**Для дипломной работы** стиль должен быть **ближе к НИР2** в экспериментальных разделах (конкретика, метрики, код) и **ближе к НИР1** в теоретических разделах (развёрнутые обзоры, глубокий анализ).

## 12. Чего КАТЕГОРИЧЕСКИ избегать

1. **Разговорный тон**: «давайте посмотрим», «попробуем», «по факту».
2. **Первое лицо**: «я», «мы» (допустимо «мы» только в крайне формальном контексте, но лучше избегать).
3. **Оценочные суждения без обоснования**: «отличная модель», «прекрасный результат». Вместо этого: «модель продемонстрировала высокие показатели (mAP@50=0.81)».
4. **Категоричность без оговорок**: не «модель лучше всех», а «модель демонстрирует наилучший компромисс между точностью и скоростью в рамках данного эксперимента».
5. **Слишком короткие абзацы**: менее 3 предложений — недопустимо для основного текста.
6. **Жирный шрифт** в тексте для выделения — не используется (только в заголовках).
7. **Emoji, восклицательные знаки, риторические вопросы** — никогда.

## 13. Шаблоны фраз для частых ситуаций

### При описании проблемы:
- «Одной из ключевых проблем является...»
- «Существенным ограничением выступает...»
- «Данный аспект создаёт дополнительные сложности для...»

### При описании решения:
- «Перспективным подходом является...»
- «Для преодоления данного ограничения предлагается...»
- «В настоящей работе используется...»

### При описании результатов:
- «Результаты эксперимента показали...»
- «Модель продемонстрировала...»
- «Достигнутые показатели подтверждают...»
- «Полученные результаты свидетельствуют о...»

### При сравнении:
- «В отличие от..., предложенный подход...»
- «По сравнению с..., данная модель обеспечивает...»
- «Среди рассмотренных архитектур наилучшие показатели продемонстрировала...»

### При переходе к следующему разделу:
- «Проведённый анализ позволяет перейти к...»
- «С учётом вышеизложенного, далее рассматривается...»
- «На основании полученных результатов, в следующем разделе...»

### При формулировке выводов:
- «Таким образом, ...»
- «В результате проведённого исследования установлено, что...»
- «Полученные данные подтверждают гипотезу о...»
