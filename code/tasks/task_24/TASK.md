## Зачем это нужно (мотивация для диплома)

FID/KID из task_23 отвечают на вопрос «насколько синтетика статистически близка к распределению реальных полевых снимков» — это мера **реалистичности**. Но есть вторая, не менее важная характеристика генеративного метода: **разнообразие внутри класса**.

Типичная слабость Neural Style Transfer — монотонность. Стилевая матрица Грама у референсного листа одна, и она накладывает одинаковый цвет, текстуру и паттерн симптомов на все контентные изображения. В результате все стилизованные кадры получаются «на одно лицо», и даже при неплохом FID модель детектора не получает нового сигнала — разнообразия в обучающей выборке не прибавляется. Диффузия, наоборот, за счёт стохастической выборки шума даёт разные варианты под один и тот же промпт и seed.

Чтобы это количественно показать в диплома, используется **LPIPS (Learned Perceptual Image Patch Similarity)**. Метрика считает перцептивное расстояние между парой изображений через feature maps предобученной сети (AlexNet или VGG), откалиброванные на датасете человеческих оценок сходства. Усреднённое LPIPS по случайным парам внутри одного класса — это оценка **внутриклассового разнообразия** синтетического набора.

Гипотеза диплома: у диффузии $\overline{\text{LPIPS}}$ близко к таковому у реальных снимков, у NST — значительно ниже (монотонность). Эта метрика закрывает пробел между FID/KID (реализм) и downstream mAP@50 (практическая польза): если NST даёт нормальный FID, но низкий LPIPS, это объясняет, почему прирост от `aug_nst` маленький — модель видит по сути одну и ту же картинку в десятке обёрток.

## Цель

Посчитать среднее LPIPS между случайными парами изображений внутри одного метода (diffusion, NST v1, NST v2, real) — для каждого из 4 редких классов отдельно и в среднем. Проверить гипотезу «diffusion ≥ real ≥ NST» и зафиксировать числа для таблицы в диплома.

3. **Инициализировать LPIPS**:
   ```python
   import lpips, torch
   loss = lpips.LPIPS(net='alex').cuda()  # alex быстрее, этого хватит для сравнения
   ```

4. **Для каждого (метод × класс)** посчитать среднее LPIPS:
   ```python
   def mean_lpips(image_paths, n_pairs=500):
       import random
       pairs = [random.sample(image_paths, 2) for _ in range(n_pairs)]
       distances = []
       for p1, p2 in pairs:
           x1 = load_and_normalize(p1)  # → тензор [-1,1], (1,3,H,W), resize 256
           x2 = load_and_normalize(p2)
           d = loss(x1.cuda(), x2.cuda()).item()
           distances.append(d)
       return np.mean(distances), np.std(distances)
   ```
   - n_pairs = 500 достаточно для надёжной оценки; больше = стабильнее, но дольше.
   - Если изображений в классе < 10 — n_pairs уменьшить до `C(n,2)` и считать по всем парам.
   - Resize до 256×256 обязателен (LPIPS обучен на этом разрешении).

5. **Также посчитать общее LPIPS** для каждого метода — по всем изображениям метода вперемешку, 2 000 случайных пар. Это даёт агрегированное число для итоговой таблицы.

6. **Опционально** — посчитать LPIPS между **synthetic и ближайшим реальным изображением** (nearest-neighbor LPIPS). Слишком маленькое значение NN-LPIPS — признак копирования, слишком большое — признак ухода из домена. Оставить на случай, если будет время.

## Что точно НЕ нужно

- Не считать LPIPS между real и synthetic — это та же задача, что FID, только другой способ. LPIPS здесь мерит именно разнообразие **внутри** одного набора.
- Не считать LPIPS на VGG — AlexNet достаточно, разница между ними меньше 5 %, а VGG в 3–4 раза медленнее.
- Не зацикливаться на идеальном разделении real-изображений по классам — если изображение имеет несколько bbox разных классов, использовать преобладающий по площади или по количеству, это приближение нормально.
