# Task 02: Конвертация Label Studio JSON → YOLO-формат + разбиение train/val/test

## Статус
pending

## Цель
Конвертировать экспорт из Label Studio (JSON с полигонами) в YOLO-формат (images/ + labels/ с .txt) и выполнить стратифицированное разбиение на train/val/test.

## Контекст
Данные лежат в одной папке — JSON-экспорт из Label Studio. Разметка в виде полигонов, нужно конвертировать в bounding boxes (YOLO-формат: `class x_center y_center width height`, нормализованные 0–1). Количество классов и изображений — определить автоматически из данных (НЕ хардкодить).

## Входные данные
Путь к папке с данными будет указан пользователем. Ожидается:
- JSON-файл(ы) экспорта Label Studio
- Папка с изображениями (jpg/png)

**Важно:** перед началом работы запроси у пользователя путь к папке с данными, если он не указан.

## Шаги

### 1. Разведка данных
- Прочитать JSON-экспорт, определить структуру
- Извлечь список всех классов (label names) и их количество
- Посчитать общее число изображений и аннотаций
- Вывести в консоль сводку: сколько классов, сколько изображений, сколько аннотаций всего

### 2. Конвертация полигонов → bounding boxes (YOLO-формат)
- Для каждого полигона: вычислить min/max по x и y → bounding box
- Нормализовать координаты относительно размера изображения (0–1)
- Формат каждой строки .txt: `class_id x_center y_center width height`
- Создать файл `data.yaml` с маппингом class_id → class_name
- Сохранить маппинг классов в `code/configs/classes.yaml`

### 3. Стратифицированное разбиение train/val/test
- Пропорции: **70% train / 20% val / 10% test**
- Стратификация по классам (по преобладающему классу на изображении), чтобы редкие классы попали во все сплиты
- Использовать `sklearn.model_selection.train_test_split` с `stratify`
- Разбиение на уровне **изображений** (не аннотаций)

### 4. Сохранить результат
Структура выходных данных:
```
code/data/dataset/
├── data.yaml          # nc, names, train/val/test paths
├── train/
│   ├── images/        # .jpg/.png
│   └── labels/        # .txt (YOLO-формат)
├── val/
│   ├── images/
│   └── labels/
└── test/
    ├── images/
    └── labels/
```

### 5. Валидация
- Проверить, что каждому изображению соответствует .txt файл
- Проверить, что в каждом .txt файле class_id в допустимом диапазоне
- Проверить, что координаты bbox в [0, 1]
- Вывести финальную сводку: число изображений и аннотаций в каждом сплите

## Ожидаемый результат

- `data.yaml` с правильными путями и именами классов
- `code/configs/classes.yaml` с маппингом
- Лог конвертации с числами: сколько изображений, аннотаций, классов, размеры сплитов
