# Task 15: CGFM в YOLOv12 — основной метод (и комбинация CGFM + Late Fusion)

## Статус
pending (после `task_12`, желательно после `task_13` и `task_14` для прямого сравнения)

## Цель

Реализовать и обучить ключевую конфигурацию дипломной работы — **CGFM** (Context-Gated Feature Modulation), FiLM-модуляцию признаков в neck YOLOv12 от внешнего контекстного энкодера. Это основной вклад главы 4 и всей дипломной работы.

Дополнительно — обучить комбинированный вариант **CGFM + Late Fusion**: контекст применяется дважды (в neck и в классификаторе после детекции). Это проверка гипотезы из `ART_001_CGFM.md` (конфигурация #6): даёт ли повторная подача контекста дополнительный прирост, или CGFM в neck уже «выжимает» всё.

## Общий протокол

Все гиперпараметры, датасет, формат артефактов — в `code/docs/chapter4_protocol.md`. Используется `code/data/dataset_final/`. Модули — из `code/models/chapter4/`.

Математическая модель (из `диплом/artifacts/ART_001_CGFM.md`):

$$c = E_{\text{ctx}}(x) \quad (\text{контекстный вектор})$$

$$\gamma_l = \sigma(W_\gamma^l \cdot c + b_\gamma^l), \quad \beta_l = W_\beta^l \cdot c + b_\beta^l$$

$$F'_l = \gamma_l \odot F_l + \beta_l$$

применяется на каждом уровне P3, P4, P5 neck. Для эксперимента в этой задаче **фиксируются**:
- Контекстный энкодер: `mobilenetv3_small_100` (самый лёгкий — оправдан для реал-тайм-сценария; другие варианты — в `task_16` аблации).
- Уровни: все три (P3, P4, P5).
- Обучение: end-to-end с warm-up-стратегией (см. §3.1 протокола главы 4).

## Специфика

- **Warm-up контекстного энкодера и FiLM** критически важен: без него γ-значения часто «залипают» около 0.5 (identity), и модуляция не обучается. Альтернатива warm-up — инициализация $W_\gamma$ близко к нулю (уже реализовано в task_12 `FiLMLayer`), но warm-up даёт более стабильную сходимость.
- **Контекстный энкодер получает изображение размера 224×224**, детектор — 640×640. Это разные ветви одного forward pass:
  ```python
  x_detector = preprocess(img, size=640)
  x_context  = preprocess(img, size=224)
  c = context_encoder(x_context)           # [B, 256]
  detections = detector(x_detector, c=c)   # neck применяет FiLM(c)
  ```
- **γ-визуализация для диплома**: ключевой инсайт. Для каждого уровня P3/P4/P5 строить тепловую карту «средний γ-коэффициент по каналам» для 10 тестовых файлов. Ожидается, что карты будут различны для разных условий съёмки (близкий план vs общий вид, разное освещение). Это подтверждает гипотезу о том, что контекстный энкодер выучил семантику сцены.
- **CGFM+Late: общий контекст.** Принципиально важно, что Late-Fusion в этой конфигурации использует **тот же контекстный вектор c**, что и CGFM в neck. Это означает, что мы передаём одну и ту же информацию в два разных места — и смотрим, даёт ли это прирост. Не обучать второй контекстный энкодер специально для Late.
- **Итоговая FPS-деградация**: приемлемо до 20 % от baseline (т.е. не ниже ~45 FPS, если baseline 55 FPS). Если деградация сильнее — записать в RESULT.md, это само по себе значимый результат.

### Ожидаемое качественное поведение

- **CGFM > SE, CBAM, Late** по mAP@50 и mAP@50-95 — основное утверждение работы. Ожидаемый прирост над baseline: +2–4 п.п. mAP@50, +1.5–3 п.п. mAP@50-95.
- **Precision и Recall оба растут** (в отличие от Late, который не меняет Recall) — это прямое следствие того, что CGFM меняет признаки **до** предсказания bbox.
- **На редких классах прирост выше** — именно здесь ограниченная обобщающая способность baseline сильнее всего страдает от недостатка контекстной адаптации.
- **CGFM+Late > CGFM**, но разница малая (+0.2–0.7 п.п. mAP@50) — повторная подача контекста даёт маргинальный выигрыш; основной вклад — в neck.
- **γ-тепловые карты различаются** по типу сцены: близкие планы усиливают каналы P3 (мелкомасштабные детали), общие виды — P5 (глобальный контекст). Это отдельный рисунок для диплома.

Если CGFM не превосходит SE/CBAM/Late — зафиксировать в RESULT.md и провести диагностику (проверить warm-up, форму γ-распределения — если все γ близки к 0.5, значит, модуляция не обучилась).
