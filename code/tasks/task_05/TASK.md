# Task 05: Балансировка редких классов — oversampling через аугментации

## Статус
pending

## Цель
Выровнять дисбаланс классов в тренировочном сплите путём целевого oversampling: для редких классов генерировать дополнительные аугментированные копии изображений, чтобы довести число аннотаций до целевого уровня.

## Зависимости
Выполнить после task_04 (классические аугментации). Работаем поверх `code/data/dataset_augmented/train/`.

## Входные данные
- `code/data/dataset_augmented/train/` — тренировочный сплит после классических аугментаций
- `code/data/dataset_augmented/data.yaml`

## Контекст
После task_04 общий объём данных вырос ×3, но **пропорции классов сохранились** — редкие классы по-прежнему недопредставлены. Нужен целевой oversampling именно для тех классов, которые ниже медианы по числу аннотаций.

## Шаги

### 1. Анализ текущего дисбаланса
- Подсчитать число аннотаций каждого класса в текущем train
- Определить **медиану** числа аннотаций
- Классы с числом аннотаций ниже медианы — кандидаты на oversampling
- Вычислить для каждого такого класса: сколько дополнительных копий нужно, чтобы приблизиться к медиане

### 2. Целевой oversampling
Для каждого изображения, содержащего аннотации редкого класса:
- Генерировать N дополнительных аугментаций (N зависит от дефицита класса)
- Использовать **агрессивные** аугментации (больше разнообразия):
  - ShiftScaleRotate (rotate_limit=30, scale_limit=0.2)
  - RandomResizedCrop
  - RandomBrightnessContrast (limits=0.3)
  - HueSaturationValue
  - ElasticTransform (alpha=30, sigma=5, p=0.3)
  - GridDistortion (p=0.3)
- Именование: `{orig_name}_bal{i}.jpg`

### 3. Ограничение
- Не допускать, чтобы одно изображение было повторено более 10 раз (diminishing returns)
- Если oversampling одного изображения недостаточен — перейти к следующему изображению с тем же классом
- После oversampling класс должен иметь ≥50% от медианного числа аннотаций (не обязательно точно достигать медиану)

### 4. Визуализации

**4.1 До/после балансировки**
- Grouped bar chart: число аннотаций по классам ДО и ПОСЛЕ oversampling
- Подпись разницы (в %) для каждого класса
- Сохранить: `code/results/task_05/balance_comparison.png`

**4.2 Imbalance ratio до/после**
- Два числа: max/min ДО и max/min ПОСЛЕ
- Визуализировать как простую метрику на графике
- Сохранить: `code/results/task_05/imbalance_ratio.png`

**4.3 Примеры oversampled изображений**
- Для каждого редкого класса: 1 оригинал + 3 аугментации рядом
- Сохранить: `code/results/task_05/oversampling_examples.png`
