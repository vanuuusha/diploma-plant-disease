## 1. Зачем EDA перед обучением детектора

EDA — это не формальность. Каждое число и график, полученные на этом этапе, либо обосновывают дальнейшие решения, либо подсвечивают риски:
- **Imbalance ratio** → необходимость балансировки (oversampling, class-balanced loss, focal loss).
- **Распределение размеров bbox** → выбор input resolution и anchor-sizes детектора.
- **Распределение числа bbox на кадр** → нужно ли настраивать max_detections, NMS iou_threshold.
- **Тепловая карта центров bbox** → есть ли пространственный bias (например, все симптомы у центра кадра → модель может переобучиться на позицию).
- **Разрешение снимков** → оценка вычислительной стоимости и нужна ли ресемплизация.

Всё это критично при обоснованном выборе архитектуры и гиперпараметров — это материал для раздела «Анализ датасета» главы 2.

## 3. Дисбаланс классов — ключевая характеристика

| Класс | Аннотаций | % |
|-------|-----------|---|
| Недостаток P2O5 | 4552 | 28.7% |
| Листовая (бурая) ржавчина | 2094 | 13.2% |
| Мучнистая роса | 2001 | 12.6% |
| Пиренофороз | 1984 | 12.5% |
| Фузариоз | 1620 | 10.2% |
| Корневая гниль | 1027 | 6.5% |
| Септориоз | 977 | 6.2% |
| Недостаток N | 875 | 5.5% |
| Повреждение заморозками | 737 | 4.6% |

### Ключевые метрики дисбаланса
- **Imbalance ratio (IR) = max/min = 4552 / 737 = 6.18×**
- **Коэффициент вариации (CV)**: CV = σ/μ = 1274 / 2030 ≈ **0.627**
- **Энтропия распределения** (нормированная): `H/log(N) ≈ 0.93` (1.0 = идеально равномерно)

### Интерпретация
- IR = 6.18× — **умеренный дисбаланс**. Для сравнения: в некоторых аграрных датасетах IR достигает 50–100× (COCO имеет ~15×), и там без балансировки обучать невозможно. 6.18× — в «серой зоне»: обучать можно, но редкие классы будут иметь заметно меньший recall.
- CV = 0.63 — разброс средней интенсивности в 63% от среднего. Это **не однородно**, но и не катастрофично.
- Практический вывод: нужна **частичная балансировка** (20–50% gap до медианы), но не полная (это вызвало бы переобучение).

### Почему P2O5 доминирует (4552 аннотации)
Недостаток P2O5 — физиологическое заболевание, проявляющееся на **всём листе как россыпь мелких пятен**. Один лист = 10–20 bbox. Другие болезни (ржавчина, септориоз) дают 1–3 крупных пятна на лист. При сопоставимом числе снимков (~500) P2O5 накапливает в 5× больше bbox.

Это важное замечание: **дисбаланс на уровне bbox НЕ означает дисбаланс на уровне снимков**. На уровне снимков все 9 классов представлены ~500–560 каждый. Балансировка по bbox vs по снимкам — разные стратегии, мы балансируем **по bbox**, так как именно это видит функция loss детектора.

## 4. Распределение числа bbox на изображение

Медиана = 2, mean = 3.57, max = 44, min = 1.

Гистограмма (`bbox_per_image.png`) показывает:
- ~40% кадров имеют ровно 1 bbox — это снимки крупным планом одного поражения,
- ~70% кадров имеют ≤3 bbox,
- «хвост» до 44 — массовые снимки P2O5 или Пиренофороза.

**Практический вывод**: `max_detections_per_image=100` (значение по умолчанию у YOLO) с запасом покрывает все реальные случаи — не нужно увеличивать.

## 5. Распределение размеров bbox

`bbox_sizes.png` — scatter нормализованного `width × height`, цвет по классу.

Наблюдения:
- Большинство bbox имеют относительные размеры **0.02–0.3** (т.е. 2–30% от изображения).
- Есть кластер **крупных bbox >0.5** — в основном у Недостатка N и Заморозков (эти болезни поражают весь лист одним большим bbox).
- «Мелкие» bbox <0.05 — у Фузариоза, Корневой гнили (точечные поражения).

**Практический вывод**:
- Нужна многомасштабная детекция (FPN/PAN). YOLOv10 имеет её нативно — решение архитектуры подтверждено.
- При инференсе разрешение 640×640 означает, что минимальный bbox в пикселях будет ~20×20 (640×0.03), что на грани способности детектора. Если на продакшн-датасете окажутся более мелкие объекты, потребуется повысить input resolution до 1024 или применить SAHI (Slicing Aided Hyper Inference).

## 6. Тепловая карта центров bbox

`bbox_heatmap.png` — 50×50 grid из всех центров bbox, нормированных в [0,1]×[0,1].

Наблюдения:
- Распределение **почти равномерное** с лёгким сгущением к центру кадра (что естественно — фотографы центрируют предмет съёмки).
- Нет признаков позиционного bias, который заставил бы модель искать симптомы только в одной области кадра.

**Практический вывод**: случайный кроп и flip-аугментации безопасны — не создадут систематического смещения.

## 7. Разрешение снимков

`image_resolutions.png` — scatter width × height в пикселях. Четыре основных кластера:
- Условно-«старые» мобильные (1024×768, 1920×1080) — ~15%,
- Средний диапазон (2000×1500 — 3500×2500) — ~65%,
- «Современные» DSLR (4000×3000 — 5000×3500) — ~15%,
- Высокое разрешение (6000+×4000+) — ~5%.

**Практический вывод**: при обучении YOLO приводим всё к 640×640 (стандарт) или 1024×1024 (если важны мелкие bbox). Массовый upscaling (от 450×320) или downscaling (от 6690×5344) потенциально вносит артефакты — стоит отдельно проверить, не снижают ли они качество. Это идея для будущего эксперимента.

## 8. Что показывают примеры изображений

`class_examples_grid.png` + `examples/class_<имя>.png` — по одному репрезентативному снимку на класс с нарисованными bbox. Это не только декоративная иллюстрация, но и **sanity check** разметки:
- все bbox лежат внутри снимка,
- класс соответствует содержимому (визуальная корректность),
- bbox адекватного размера.

Выборочно просмотрел все 9 — отклонений не нашёл.

## 9. Boxplot площадей bbox по классам

`bbox_area_by_class.png` в log-шкале:
- **Недостаток P2O5** — медиана ~0.002 (мелкие), очень широкий разброс.
- **Недостаток N, Заморозки** — медиана ~0.2 (крупные, лист целиком), компактное распределение.
- **Септориоз, Пиренофороз, Мучнистая роса** — средний размер ~0.01–0.05.

Классы разделяются по **типичному размеру симптома**. Это может быть использовано в постпроцессинге: если детектор предсказал класс «Недостаток N» с bbox < 0.05, это подозрительно — возможно, false positive.
